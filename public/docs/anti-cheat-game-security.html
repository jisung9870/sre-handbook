● Overview

Anti-cheat — 게임 보안의 가장 어려운 영역

치팅은 단순 부정행위가 아니라 게임 매출·retention·브랜드의 직접적 위협이다. F1·F2·F3가 토폴로지·네트워크·매치메이킹이었다면 F4는 게임 자체의 무결성을 다룬다. 퍼블리셔 SRE가 anti-cheat 도입·운영·장애 시 알아야 할 깊이. (2026-05 기준)

치팅이 왜 큰 문제인가 — 게임 사업의 직접적 위협

Retention 파괴

"매치마다 핵 만남" → 며칠 안 이탈. 신규 사용자에게 결정적. 특히 라이트 사용자 학살은 게임 죽음의 첫 신호.

매출 직접 손실

치터 자체는 결제 안 함, 정상 사용자가 떠나면 매출 폭락. F2P 게임은 retention = 매출.

경쟁 무결성

경쟁 모드·랭크가 의미 없어짐. e-sport 가치 붕괴 → IP 가치 손실.

브랜드·평판

"이 게임은 핵 천국" 평판 → 신규 출시 이전·확장 모두 영향.

2026 산업 동향

2025-09 Valve VAC Live stealth update가 사상 최대 anti-cheat crackdown. 2025년 PUBG는 BattlEye로 약 26만 DMA-기반 cheater 영구 ban. DMA cheat 탐지가 2026년 모든 솔루션의 top priority.

Anti-cheat 4대 모델

Kernel-mode (Ring 0)

OS 커널 레벨에서 동작. 다른 프로세스 접근·시스템 호출 모두 감시 가능. Vanguard·EAC 일부·BattlEye 일부.

User-mode (Ring 3)

일반 애플리케이션 레벨. 자기 프로세스만 보호 가능. 우회 쉬움. VAC·HackShield 일부.

Server-side

서버에서 통계·행동 분석. 클라 무관. ML 기반 (VACnet). 가장 안전하지만 탐지 지연.

Hybrid

위 3개 조합. 현실의 모든 상용 솔루션. 클라 보호 + 서버 검증 + 통계 분석.

퍼블리셔 SRE의 anti-cheat 관여 범위

SRE 책임

Anti-cheat 솔루션 통합·SDK 인테그레이션, 신고 백엔드, ban 시스템 운영, 어필 워크플로우, 텔레메트리, anti-cheat 장애 대응

게임사 책임

치팅 룰 정의, ban 결정 정책, in-game 신고 UI, 게임 코드의 anti-tamper

공동 (회색지대)

대규모 ban wave 시점·범위, 신규 cheat 탐지 후 대응, 어필 처리 SLA, anti-cheat 솔루션 선택·계약

★ 솔루션 선택은 정책
EAC·BattlEye·Vanguard 같은 외부 솔루션 도입은 (1) 라이선스 비용 (2) 사용자 신뢰 영향 (3) 플랫폼 호환성 (4) 게임 내 개발 부담 — 4가지 정책 결정. SRE가 데이터 (호환성·성능·계약 비교표) 제공.
● Threats

치팅 종류 & 공격 벡터

각 치팅의 동작 원리·탐지 신호·완화 방법. 어떤 치팅이 가능한지 알아야 어느 layer에서 막을지 결정 가능. 2026년 핵심 위협은 DMA 치트 — 별도 섹션.

Aimbot · Wallhack · ESP — FPS의 영원한 적

Aimbot

마우스 자동 조준. 메모리에서 적 위치 읽고 마우스 입력 시뮬레이션. Triggerbot은 발사만 자동, Recoil control은 반동 자동 보정.

Wallhack

벽 너머 적 표시. 렌더링 후킹 (DirectX·Vulkan)이나 메모리에서 적 위치 직접 읽음.

ESP (Extra Sensory Perception)

적 정보 화면 표시 — 체력·거리·이름·박스 그리기. Wallhack 변형, 외부 오버레이로 구현 일반.

Radar

외부 화면에 미니맵 표시. 외부 도구 (laptop)에서 패킷 분석해 적 위치 표시. 게임 자체 안 건드려서 탐지 어려움.

★ 탐지의 어려움
Wallhack·ESP·Radar는 수동적 (정보 읽기만) 이라 클라 행동에 흔적이 적다. Aimbot은 마우스 trajectory 분석으로 탐지 가능 — VACnet 같은 ML 모델의 강점 영역.

Speedhack · Teleport · Fly hack

Speedhack

이동 속도 증가. 시간 함수 후킹 (QueryPerformanceCounter 등) → 게임이 더 많은 frame 처리. 서버 측 속도 검증으로 차단 가능.

Teleport hack

좌표 직접 변경. 서버가 한 tick에 비현실적 거리 이동 감지하면 차단. 기본 검증으로도 막힘.

Fly hack

중력 무시 좌표. 서버 물리 검증 (땅에 발 닿아야 함)으로 차단.

No-clip

벽 통과. 서버 collision 검증으로 차단.

★ Server-authoritative의 효과
이 종류는 서버가 권위적 시뮬레이션을 하면 대부분 차단. F1에서 다룬 server-authoritative 모델이 anti-cheat 1차 방어선. 클라가 보낸 입력의 정합성 검증만 잘해도 차단 효과 큼.

메모리 변조 — Cheat Engine 등

프로세스 메모리에서 게임 데이터 (체력·자원·좌표) 찾아 직접 수정. Cheat Engine이 표준 도구. 단순 캐주얼 게임은 매우 쉬움.

공격 도구

Cheat Engine, ArtMoney, ReClass.NET 등. 무료·오픈소스. 사용법 유튜브 다수.

방어 1: Memory protection

중요 변수를 의도적으로 읽기 어렵게 (XOR encrypt, pointer chain). 우회 가능하지만 시간 소요.

방어 2: Server validation

중요 데이터 (체력·자원)는 서버가 source of truth. 클라가 변조해도 서버 검증에서 거부.

방어 3: Anti-debug

디버거 attach 탐지·차단. 메모리 분석 자체를 어렵게.

패킷 변조

공격 1: 데이터 변조

송수신 패킷 가로채 내용 변경. 클라가 "데미지 100" 보내야 하는데 "1000"으로 변조.

공격 2: Replay

정상 패킷 (예: "보스 처치 보상") 캡처 후 반복 송신.

공격 3: Drop / delay

특정 패킷 의도적 drop 또는 delay → lag switch 어뷰징.

방어

F2에서 다룸 — 패킷 암호화 (DTLS·GNS), HMAC 서명, sequence number, server-authoritative.

DMA 치트 — 2026년 핵심 위협

두 PC 사용. 게임 PC에 DMA 카드 (PCIe 카드) 꽂고, 두 번째 PC가 그 카드 통해 게임 PC 메모리 직접 읽음. 게임 PC에서는 cheat 프로세스가 안 돌므로 anti-cheat가 탐지하기 매우 어려움.

왜 위험한가

cheat가 게임 PC 외부에 있음. 모든 software 기반 anti-cheat 우회. ESP·Radar·data 읽기 자유로움.

탐지 방법 1

DMA 카드의 하드웨어 시그니처 탐지 (PCIe device ID 등). 우회 가능 (ID 변조).

탐지 방법 2

비정상 메모리 접근 패턴 탐지. Vanguard·EAC·BattlEye 모두 이 방향 강화 중.

탐지 방법 3

Server-side 행동 분석으로 우회. cheat가 어디 있든 게임 행동 자체는 비정상이므로 ML로 탐지.

★ 산업 대응 (2026)
BattlEye는 2025년 PUBG에서 약 26만 DMA cheater 영구 ban, 2026년 top priority로 DMA 탐지 강화 선언. EAC·Vanguard도 동일 방향. 핵 산업이 DMA로 이동 중 → 모든 anti-cheat가 hardware-level 탐지·HWID ban 강화.

봇 · 매크로 · auto-clicker

Farming bot

MMORPG에서 자동 사냥·자원 수집. 24/7 가동. 인게임 경제 파괴 (게임 화폐 RMT).

매크로

특정 키 자동 입력. 격투의 콤보, MMO의 스킬 사이클. 단순 도구 = 사용자 도구, 정교 = 봇.

Click farming

광고·이벤트·일일 미션 자동 수행. 보상만 챙기고 게임 안 함.

탐지

입력 패턴 다양성 (사람은 자연스러운 변동, 봇은 일정), 24/7 활동 시간, 비정상 KDA·이동 trace 분석. ML 분류.

핵 판매 생태계

2020년대 들어 cheat이 산업화. 월 구독 모델 (월 $30~100) 일반. cheat 제작자·유통자·고객까지 명확한 시장.

구독 모델

월정액 cheat. private cheat (소수 사용자에게만 판매)이 탐지율 낮음 → 비쌈.

HWID Spoofer

HWID ban 우회 도구. cheat과 함께 판매. anti-cheat의 HWID ban 무력화.

법적 대응

BattlEye·Riot·Bungie 모두 cheat 제작자 상대 소송. 일부 승소·합의로 cheat 산업 일부 위축.

플랫폼 책임

cheat 판매 사이트의 결제 처리 (Stripe·PayPal) 차단, 광고 플랫폼 (YouTube·Google) 정책 강화.

● Solutions

Anti-cheat 솔루션 깊이 비교

VAC · EAC · BattlEye · Vanguard · 한국 솔루션 (HackShield · nProtect · Xigncode) · 콘솔 정책. 각 솔루션의 동작 원리·강약점·라이선스·도입 함정.

VAC + VACnet (Valve)

VAC (Valve Anti-Cheat)

2002 출시. Steam 게임의 표준 anti-cheat. 시그니처 기반 + delayed ban. 실시간 차단 약함.

VACnet (현재 VAC Live)

Server-side ML. CS:GO/CS2 매치 데이터로 학습. aim trajectory·반응 시간·움직임 패턴 분석. Real-time ban 가능.

VACnet 3.0 (2026)

Real-Time Bans + AI-powered detection + kernel-level security 추가. 2025-09 stealth update가 사상 최대 cheater 차단.

강점·약점

강점: 무료, Steam 통합, ML 진화. 약점: kernel-level 약함, 시그니처 회피 가능, 차단율 비판.

Easy Anti-Cheat (EAC) — Epic 운영

운영자

2018 Epic Games 인수. Epic Online Services (EOS) 일부로 통합. 무료 (Epic 게임 한정), 외부 게임은 라이선스.

모드

Hybrid (kernel + user). 부팅 시 driver. 비교적 가벼움.

Linux/Steam Deck (2026)

EOS 버전은 toggle 한 번에 활성화, Wine/Proton 호환. Rocket League 2026-04-30부터 EAC required, Steam Deck 호환 유지. 구버전 non-EOS는 별도 작업.

대표 적용

Fortnite, Apex Legends, Rust, Dead by Daylight, Rocket League. 가장 광범위한 적용 솔루션.

BattlEye

운영자

BattlEye GmbH (독일). 2004 출시. 라이선스 모델.

특징

Kernel-mode driver, 서버 측 검증 결합. "The Anti-Cheat Gold Standard" 자칭.

대표 적용

PUBG · Rainbow Six Siege · DayZ · Arma · Fortnite (병행 사용 시기). 한국·일본 게임 일부.

2026 전략

DMA cheat 탐지 top priority, hardware-based re-entry block 강화. 2025 PUBG에서 약 26만 DMA cheater 영구 ban. cheat 유통 네트워크 법적 대응.

Riot Vanguard

운영자

Riot Games 자체 개발. 2020 Valorant launch와 함께 등장.

모드

Ring 0 (kernel) custom anti-cheat. 부팅 시 driver 시작 (system boot에 함께), 게임 클라 실행 시 active monitoring.

적용 게임 (2026)

League of Legends · TFT · 2XKO · Valorant 모두 적용. Riot 게임 전체로 확산.

2026 보안 이슈

2026-01 critical vulnerability 발견 — 물리 접근 + malicious driver 설치 가능. Riot 패치 완료, 실 exploit 증거 없음.

★ 강력함과 논쟁
Vanguard는 가장 강력한 PC anti-cheat 중 하나. 동시에 가장 논쟁적 — 부팅 시 driver, kernel 권한, passive 모니터링이 사용자 신뢰·privacy 우려. 자세한 논쟁은 다음 탭 참조.

● Kernel vs User

Kernel mode vs User mode 깊이

Anti-cheat가 어느 레벨에서 동작하느냐가 강력함과 논쟁 모두를 결정한다. Ring 0 (kernel)이 강력하지만 사용자 시스템 영향·privacy 우려가 동시에 큼.

x86 Ring 모델 (Privilege Levels)

Ring 0Kernel mode — 모든 시스템 자원 접근. driver·OS 핵심
Ring 1·2현대 OS 거의 안 씀
Ring 3User mode — 일반 애플리케이션. 자기 자원만 접근
★ 의미
Ring 0에서 동작하는 anti-cheat는 다른 모든 프로세스를 감시할 수 있다. 메모리 접근·시스템 호출·파일 시스템 모두 제한 없음. 사용자 자신보다 더 강한 권한이 게임에 부여됨.

Kernel mode 장단점

장점 1: 강력한 탐지

다른 프로세스 (cheat) 메모리 접근, 후킹 탐지, driver-level cheat 차단 가능.

장점 2: Cheat과 동등 무기

현대 cheat은 kernel driver 형태가 일반. user mode anti-cheat는 kernel cheat을 못 보지만 kernel anti-cheat는 가능.

단점 1: 시스템 안정성

Driver bug → 블루스크린·시스템 크래시. 게임 종료해도 driver 잔존 시 PC 재부팅 필요.

단점 2: 보안 위험

Driver 자체 vulnerability → kernel privilege 공격 진입점. 2026-01 Vanguard vulnerability 사례.

단점 3: Privacy

모든 프로세스·파일 접근 가능. 개인정보·다른 게임·업무 환경 모두 anti-cheat에 노출.

단점 4: 사용자 신뢰

"내 컴퓨터에 게임사 driver를 boot 시 동작" 거부감. 보안 의식 높은 사용자 이탈.

User mode 장단점

장점 1: 안전·낮은 위험

크래시해도 게임만 영향. 시스템 안정성 ↑. 보안 vulnerability 영향 작음.

장점 2: Privacy

자기 프로세스만 접근. 다른 시스템 자원 노출 없음.

장점 3: 사용자 거부감 낮음

일반 애플리케이션과 다를 바 없음.

단점 1: 약한 탐지

Kernel cheat 우회 불가능. 외부 프로세스 cheat 탐지 어려움.

단점 2: 자기 보호 한계

다른 프로세스가 anti-cheat 자체를 종료·우회할 수 있음.

단점 3: DMA 무력

외부 PC가 메모리 직접 읽는 DMA cheat에는 user mode anti-cheat 무력.

부팅 시 driver 로드 — Vanguard 모델

Vanguard·일부 EAC가 채택. 게임 실행 전부터 시스템에서 active. 시스템 boot 시 함께 로드 → 게임 종료 후에도 일부 driver 잔존.

왜 부팅 시?

일부 cheat은 OS 부팅 후 anti-cheat 시작 전 사이에 활성화. 부팅 시 anti-cheat이 함께 시작하면 cheat이 anti-cheat 우회 불가능.

강점

현재 가장 강력한 PC anti-cheat. Riot Vanguard가 표준.

사용자 우려

"게임 안 할 때도 driver 동작?" → 일부 사용자 이탈.

대응

Vanguard는 게임 실행 안 할 때는 passive 모니터링만 (active 검사 X). 명시적 안내·privacy policy 공개로 신뢰 회복 시도.

Vanguard 논쟁 — privacy vs security

Riot Vanguard가 2020 Valorant launch와 함께 도입되며 시작된 논쟁. 2024년 LoL 확대 적용 시 다시 폭발. 2026년에도 진행 중.

Riot 입장

"강력한 anti-cheat 없이는 경쟁 게임 무결성 불가능. 데이터 수집은 anti-cheat 목적만, 외부 송신 안 함." Privacy & Security 팀 메시지 공식 발표.

비판 입장

"Kernel 권한은 위험. 2026-01 vulnerability가 증명. 게임사를 어떻게 신뢰?" 보안 연구자·privacy 옹호자 우려.

실 영향

일부 사용자 이탈, 일부는 신경 안 씀. LoL 확대 적용 후 retention 감소 명확하지 않음.

업계 trend

2020년대 들어 강한 anti-cheat 수요 증가 → kernel mode 표준화 흐름. 단 Linux/Mac 호환성과 충돌.

★ SRE 입장
Anti-cheat 솔루션 선택은 사업 결정. SRE는 (1) 솔루션별 사용자 거부율·이탈률 데이터 (2) 호환성·성능 영향 (3) 사고 시 대응 계획 — 데이터로 정책 결정 지원. 한쪽 입장 옹호 X.

● Server-side Detection

Server-side Detection — 클라 무관 안전 영역

클라 anti-cheat는 항상 우회 가능. 서버 측 통계·행동 분석은 cheat이 아무리 정교해도 비정상 행동을 남긴다. ML 기반 탐지가 핵심 트렌드.

통계 이상치 — KDA·헤드샷률·반응시간

KDA

해당 tier 평균 대비 비현실적 KDA. 평균 1.2 tier에서 5.0+ 지속이면 의심. 단 절대값보다 추세·분포 중요.

헤드샷률

FPS에서 헤드샷 비율 80%+ 지속이면 aimbot 의심. 정상 인간 한계 50~60%.

반응 시간

적 발견 → 발사 reaction time. 100ms 미만 지속이면 비정상 (인간 평균 200~300ms).

조준 정확도 분포

일반 사용자는 조준 정확도가 거리·상황 따라 변동. aimbot은 일정. 분포 분석.

★ False positive 위험
고숙련 e-sport 선수도 헤드샷률 70%+ 가능. 통계 이상치만으로 ban은 위험. 다른 신호 (행동 패턴·신고)와 결합해 결정.

ML 기반 탐지 — VACnet 사례 깊이

VACnet (현 VAC Live)이 deep learning anti-cheat의 대표. CS:GO·CS2 매치 데이터를 기반으로 학습된 신경망이 실시간 매치 중 cheat 패턴 탐지.

aim trajectory·반응 시간·움직임 패턴·입력 timing. 수백만 매치 데이터.

강점

새로운 cheat에도 대응 (시그니처 X). 클라 무관. 우회 어려움.

약점

탐지 지연 (실시간 한계), false positive 위험, 모델 재학습 비용.

2026 발전

VACnet 3.0이 real-time ban + AI detection 강화. 2025-09 stealth update가 사상 최대 cheater 차단.

행동 패턴 — 마우스 trace 분석

인간 마우스 movement

곡선·미세 jitter·overshoot. 가속·감속이 자연스러움. 손가락 떨림 흔적.

Aimbot trace

직선·정확한 가속·즉시 정지. 너무 정확. ML이 인간 vs aimbot 분류.

Smooth aimbot

고급 aimbot은 인간 흉내내려고 약간의 jitter·delay 추가. 더 정교한 ML 필요.

적용

VACnet, Riot Vanguard, EAC 모두 이 방향 분석. 실 cheat 데이터를 학습 데이터로 사용.

멀티 계정 fingerprint

HWID (Hardware ID)

CPU·GPU·motherboard 시리얼 조합. ban 시 hardware 차단. HWID spoofer로 우회 가능.

IP 주소

같은 IP에서 여러 계정 → 의심. CGNAT·VPN 우회 가능.

행동 fingerprint

마우스 movement·키 입력 timing·이동 경로 — 사람마다 고유. ML로 동일 사용자 추론.

Steam·Epic·콘솔 계정 ID 추적. 결제 정보 (카드·주소) 매칭.

신뢰 점수

신고자의 신뢰도 (과거 신고 정확도)에 가중치. 무작위 신고는 영향 적음.

다수 신고

같은 매치 다수 신고 → 자동 검토 큐 진입. ML 분석 + 사람 리뷰.

리플레이 검토

신고된 매치 리플레이 자동 분석. 의심 시 사람 리뷰. e-sport 운영 도구.

대규모 Ban Wave 운영

즉각 ban이 아니라 일정 기간 (1~2주) 의심 사용자 데이터 축적 → 한 번에 대규모 ban. cheat 제작자가 무엇이 탐지됐는지 파악 어렵게 함.

왜 wave 방식?

즉각 ban → cheat 제작자가 즉시 우회 (탐지된 패턴 학습). 지연 ban → 우회 학습 어려움.

VAC 패턴

VAC은 전통적으로 delayed ban (수일~수주 지연). 단 VACnet은 real-time ban도 가능.

EAC·BattlEye

실시간 + wave 혼합. 명확 cheat은 즉시 kick, 의심은 wave로.

발표

Wave ban 후 공식 발표 (몇 만 명 ban). 커뮤니티 신뢰 회복·deterrent 효과.
