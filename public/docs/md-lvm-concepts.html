● md / Software RAID

md — Multiple Device

커널 레벨에서 여러 block device를 하나의 RAID 장치(/dev/mdX)로 묶는 Linux Software RAID.

개요

핵심 역할

여러 물리 디스크를 하나의 논리 장치로 묶어 데이터 보호(redundancy)와 성능 향상을 제공. 전용 RAID 카드 없이 커널이 직접 처리.

관리 도구

mdadm으로 RAID 배열 생성·조회·복구. 상태는 /proc/mdstat에서 실시간 확인. 설정은 /etc/mdadm/mdadm.conf에 보존.

● LVM

LVM — Logical Volume Manager

물리 디스크를 풀(Pool)로 묶은 뒤 논리적으로 슬라이싱. 파티션 크기를 런타임에 자유롭게 조절할 수 있는 추상화 계층.

개요

핵심 역할

물리 파티션의 고정된 크기 한계를 넘어 동적 볼륨 관리를 가능하게 한다. 스냅샷, 씬 프로비저닝, 온라인 리사이즈 같은 유연한 운영이 가능하다.

핵심 장점

파티션 리사이즈, 볼륨 스냅샷(lvcreate --snapshot), 씬 프로비저닝, 디스크 핫 추가를 지원한다. 물리 파티션만으로는 어려운 유연한 운영이다.

PV (Physical Volume)

LVM이 관리하는 최소 단위. 물리 디스크나 파티션, 혹은 /dev/mdX RAID 배열도 PV가 될 수 있다. pvcreate로 초기화.

VG (Volume Group)

PV들을 하나로 합친 스토리지 풀. PE(Physical Extent) 단위로 공간 관리. vgcreate로 생성, vgextend로 PV 추가.

LV (Logical Volume)

실제 파일시스템을 올리는 논리 볼륨. VG에서 원하는 크기만큼 잘라낸다. lvextend / lvreduce로 온라인 리사이즈. mkfs로 파일시스템을 만들고 마운트.

LVM 명령어

Command설명
pvcreate /dev/sda1PV 초기화
pvsPV 목록 간략 조회
pvdisplay /dev/sda1PV 상세 정보
pvmove /dev/sda1PV 데이터를 다른 PV로 마이그레이션
Command설명
vgcreate vg0 /dev/sda1 /dev/sdb1VG 생성 (PV 여러개 묶기)
vgsVG 목록 간략 조회
vgextend vg0 /dev/sdc1VG에 PV 추가
vgreduce vg0 /dev/sda1VG에서 PV 제거
Command설명
lvcreate -L 50G -n lv_data vg050G LV 생성
lvextend -L +20G /dev/vg0/lv_dataLV 20G 확장
lvreduce -L -10G /dev/vg0/lv_dataLV 10G 축소 (주의!)
lvcreate --snapshot -L 5G -n snap0 /dev/vg0/lv_data스냅샷 생성
lvsLV 목록 간략 조회
● md + LVM Combined

💡 TIP
프로덕션 서버에서는 md(RAID)로 디스크 장애에 대비하고, 그 위에 LVM을 올려서 볼륨 관리 유연성을 확보하는 것이 표준 패턴이다.
