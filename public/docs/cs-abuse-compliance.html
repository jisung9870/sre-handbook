● Overview

CS · 어뷰징 · 컴플라이언스의 통합 관점

세 도메인은 분리된 팀처럼 보여도 실제로는 같은 "신뢰와 안전(Trust & Safety)"의 세 측면이다. 사용자 문의(CS)는 어뷰징 신호의 1차 채널이고, 어뷰징 처리는 컴플라이언스 의무가 따라붙으며, 컴플라이언스 위반은 곧 CS 폭주로 이어진다.

3 도메인의 관계

CS·어뷰징·컴플라이언스는 한 팀에 속하지 않더라도 운영적으로 단단히 연결되어 있다. 한 영역의 누락은 다른 영역의 폭발로 돌아온다.

CS → 어뷰징

"내 아이템이 사라졌어요"는 계정 탈취 신고의 최전선. CS 1차 응대가 패턴을 일찍 발견하면 ATO 캠페인을 빨리 차단할 수 있다. CS 티켓 텍스트가 어뷰징 탐지의 핵심 input.

어뷰징 → CS

대규모 제재 캠페인은 억울한 사용자 문의 폭주를 부른다. CS팀에 정지 사유 코드·증거 요약을 미리 공유하지 않으면 1차 응대가 마비된다.

컴플라이언스 → CS

확률 표시 의무 위반 → 손해배상 청구 + CS 문의 폭주. 법령 변경 시점에 가장 먼저 부담을 받는 건 CS 라인. 법무·CS·플랫폼이 사전에 답변 시나리오를 합의해야 한다.

컴플라이언스 → 어뷰징

EU DSA, AI Act 등은 제재 시 사용자 권리(이의제기·구제)를 보장한다. 어뷰징 시스템 설계가 처음부터 "이의제기 절차 + 인간 검토"를 전제하지 않으면 위법이 된다.

★ KEY
한 도메인의 정책 결정은 항상 다른 두 도메인에 영향을 미친다. 어뷰징팀이 "수만 명 자동 정지" 결정을 단독으로 내리면 CS가 폭주하고 컴플라이언스에 걸릴 수 있다. 3자 협의 SOP가 운영 안정성의 핵심.

핵심 KPI

3 도메인 각각의 지표를 한 대시보드로 묶어 본다. 한 KPI 악화가 다른 도메인의 신호인 경우가 많다.

CS — First Response Time

티켓 첫 응답까지의 중앙값. 표준 SLA: 일반 24h, 결제 4h, 보안 1h. 중앙값보다 P95가 더 중요 — 긴 꼬리가 사용자 분노의 원천.

CS — Resolution Rate

티켓이 1차에서 종결되는 비율. 셀프서비스 성공률·FAQ 적합도의 대리 지표. 50%대 → 70%대로 끌어올리면 CS 인력 부담이 절반.

어뷰징 — Detection Precision

자동 정지 결정의 정밀도(사후 이의제기 인용률 역수). 거짓 양성(false positive)이 많으면 무고한 사용자가 정지되어 CS 폭주.

어뷰징 — Time to Detect

어뷰징 발생 → 자동 탐지까지의 중앙값. 매크로·봇은 분 단위, RMT 캠페인은 시간 단위. 탐지 지연이 곧 피해 누적.

컴플라이언스 — 위반 0건

"위반 0건"은 KPI라기보단 게이트. 월간 리뷰에서 위반·잠재 위반이 발견되면 즉시 패치. 0건이 한 번 깨지면 신뢰 회복에 분기 단위가 든다.

컴플라이언스 — 감사 대응 시간

규제기관 자료 요청에 응답하는 시간. EU GDPR은 30일, 한국 개인정보위는 사안별. "자료 어디 있는지 모름"이 흔한 실패 모드 — 평소 인덱싱이 핵심.

● CS Operations

CS · 운영

고객지원은 단순 상담이 아니라 플랫폼의 청취 채널 + 어뷰징 탐지의 1차 신호 + 컴플라이언스 증거 수집의 통합 시스템이다. 잘 설계된 CS는 사고를 막고, 부실한 CS는 사고를 키운다.

티켓 시스템 아키텍처

대부분 퍼블리셔는 SaaS 티켓 플랫폼(Zendesk, Freshdesk, Helpshift)에 자체 백엔드를 결합해 쓴다. 게임 데이터·인증·결제 정보가 자동 연동되어야 1차 응대가 빠르다.

SaaS 티켓 플랫폼

· Zendesk: 글로벌 표준, 다국어·다채널·자동화 강력
· Freshdesk: 가성비, 미드 사이즈에 인기
· Helpshift: 모바일 게임 SDK 특화, 인앱 채팅
· 자체 빌드: 대형 퍼블리셔(엔씨·넥슨급)는 자체 시스템 운영

인앱 SDK 통합

모바일 게임은 인앱에서 바로 문의 가능해야 한다. SDK가 현재 사용자 ID·디바이스·게임 상태·최근 결제 영수증을 자동 첨부 → CS 응대 컨텍스트가 미리 채워짐.

게임 데이터 연동

티켓 화면에서 GM이 사용자 캐릭터 · 인벤토리 · 결제 내역 · 정지 이력을 한 번에 조회. 게임 백엔드 API와 SaaS 플랫폼 사이의 connector가 핵심.

티켓 라우팅

유형(결제·계정·버그·신고·일반)·언어·우선순위로 자동 분배. 결제·보안은 별도 큐 + 단축 SLA. 일반 큐와 섞어두면 긴급 건이 묻힌다.

FAQ · 셀프서비스 자동화

티켓을 줄이는 가장 큰 레버는 사용자가 스스로 해결하게 만드는 것. 잘 만든 FAQ + 셀프서비스 도구는 CS 인력을 1/3로 줄인다.

FAQ 카테고리화

계정·로그인·결제·게임플레이·기술이슈·신고·기타. 각 카테고리에 "자주 묻는 5개"가 인앱·웹 첫 화면에 노출되어야 한다.

셀프서비스 도구

· 비밀번호 재설정 (이메일·SMS·OTP)
· 계정 복구 (소셜 로그인 재연동)
· 결제 내역 조회 (영수증·환불 상태)
· 미수령 아이템 재청구 (결제는 됐는데 못 받은 경우 자동 복구)

챗봇 · LLM 분류

질문을 LLM으로 분류해 적절한 FAQ로 안내. 해결 안 되면 사람 상담사로 자동 전환. 챗봇만으로 해결률 30~50% 가능. 단, LLM 환각 위험이 있어 결제·계정복구는 반드시 사람.

다국어 자동 번역

들어오는 티켓을 사내 응대 언어로 자동 번역, 답변을 사용자 언어로 자동 번역. 품질 검증 없이 그대로 보내면 사고 — 결제·정지처럼 민감한 답변은 사람이 검토.

★ TIP
"해결 안 됐어요" 버튼이 FAQ에 항상 있어야 한다. FAQ 클릭 → 해결 안 됨의 비율을 추적하면 어느 FAQ가 부족한지 정확히 보인다. 이 지표가 셀프서비스 개선의 가장 확실한 입력.

표준 워크플로우 — 환불 · 복구 · 정지

CS의 핵심 작업 3가지. 각각 표준 절차·결정권자·증거 보존 규칙이 정해져야 한다.

유형표준 절차결정권증거 보존
결제 환불영수증 검증 → 미수령 여부 확인 → 스토어 환불 트리거 또는 게임 내 보상금액별 (소액=1차 / 중액=리드 / 고액=매니저)영수증·로그·CS 기록 7년 (세무·전자상거래법)
계정 복구본인 확인(이메일·휴대폰·결제) → 디바이스 검증 → 비번 재설정·소셜 재연동본인 확인 통과 시 자동, 의심 시 보안팀복구 절차 로그·승인자·복구 시점
아이템 복구로그 확인(획득·소비·소실) → 정상 손실 vs 시스템 오류 판별 → 복구 또는 대체 보상1차 GM (한도 내) / 한도 초과 시 매니저로그·복구량·승인자
계정 정지증거 패키지(로그·신고) → 1차 GM 검토 → 2차 매니저 승인 → 통보·이의제기 안내매니저급 이상 (자동 정지는 별도)정지 사유·증거·통보 시점·이의제기
이의제기 처리접수 → 다른 GM 재검토(맨처음 결정자 X) → 인용/기각 → 사용자 통보리드급 이상이의 사유·재검토 결과·통보
⚠ PITFALL
"규정에 없는 케이스"가 매주 새로 나온다. 예외 처리 권한·기록 의무를 명확히 하지 않으면 (1) GM이 자기 재량으로 보상을 남발 (2) 비슷한 케이스가 일관성 없이 처리 (3) 외부 분쟁 시 근거 부족. "예외는 매니저 승인 + 기록 + 분기 리뷰"가 표준.

GM 툴 · 권한 · 감사로그

GM(Game Master) 도구는 운영의 칼날이자 가장 큰 보안 위험. 한 명이 데이터를 다 볼 수 있고 아이템을 무한 지급할 수 있다면, 내부 사고가 외부 사고보다 훨씬 크다.

최소 권한 원칙

GM마다 권한 등급(L1·L2·L3·매니저)을 분리. L1은 조회 + 표준 보상만, L2는 정지·아이템 지급, L3는 결제 환불. 매번 권한 검토 + 분기 재인증.

감사로그 (Audit Log)

모든 GM 액션은 누가·언제·무엇을·왜를 기록. 사유 없는 액션은 차단. 매니저 임의 편집 불가(append-only). 로그 자체가 중요한 컴플라이언스 자산.

위험 액션 2-eyes

아이템 대량 지급·계정 정지·결제 환불 같은 고위험 액션은 다른 GM의 승인이 있어야 실행. 한 명이 단독으로 못 끝낸다.

이상 행동 탐지

GM 본인이 어뷰징 대상이 될 수 있다. "한 GM이 1시간에 100건 보상"은 자동 알림 + 매니저 검토. 내부 사고는 외부보다 영향이 훨씬 크다.

GM 권한 매트릭스 예시
# 권한 등급 정의 (예시)
L1 신입: read_user, send_template_response, small_item_grant(<50K)
L2 일반: + large_item_grant(<500K), temp_ban(<7d), restore_account
L3 시니어: + refund_payment(<1M), perm_ban(2-eyes 필요)
Mgr 매니저: + override_decision, audit_review, policy_exception

# 모든 액션 = audit log + 사유 + 영향 사용자 ID

다국어 · 시차 운영

글로벌 게임은 24/7 다국어 CS가 필요하다. 모두 정직원으로 채우면 비용 폭발 → BPO·자동 번역·follow-the-sun 모델의 조합이 표준.

Follow-the-sun

한국·일본 일과시간은 한국·일본 CS, 유럽 시간대는 EU CS, 미주 시간대는 NA CS. 티켓이 시차 따라 자동 라우팅. 같은 사람이 새벽 3시 응대 안 함.

BPO 활용

주력 언어(KR·JP·EN)는 사내, 마이너 언어(DE·FR·ES·PT·AR)는 BPO. 표준 답변 템플릿·게임 도메인 용어 사전 제공이 BPO 품질의 결정 요인.

자동 번역 + 사람 검수

일반 응대는 자동 번역, 결제·정지·법적 분쟁은 사람 검수. "번역기 그대로 발송"이 사고로 이어진 사례 多 — 특히 일본·아랍어 같은 고context 언어.

현지화 ≠ 번역

한국·일본은 정중한 사과 표현 필수, 영미권은 직접적 해결책 우선, 유럽은 GDPR 권리 안내. 같은 답변을 그대로 번역하면 어색하거나 무례해 보임.

● Abuse & Security

어뷰징 · 보안

게임 어뷰징은 일반 웹 보안의 문제(계정 탈취·결제 사기) + 게임 도메인 특수 문제(매크로·작업장·RMT)의 조합이다. 탐지·제재·이의제기의 파이프라인이 컴플라이언스를 만족하면서도 빠르게 작동해야 한다.

어뷰징 분류 매트릭스

어뷰징은 크게 4 영역으로 나뉜다. 각각 탐지 시그널·제재 도구·관련 컴플라이언스가 다르다.

① 계정 어뷰징

· 계정 탈취(ATO), 크리덴셜 스터핑
· 자동 계정 생성(봇 가입)
· 다중 계정·차명 계정
· 신원 도용·미성년 우회
탐지 시그널: 로그인 패턴, 디바이스 핑거프린트, IP 평판

② 결제 어뷰징

· 도용 카드(stolen card)
· chargeback 후 환불 받고 아이템 보유
· 환불 정책 악용 (Steam 2시간 룰 매번 사용)
· 가격 조작·국가별 차익 거래
탐지 시그널: 결제 패턴, BIN, 디바이스, 환불율

③ 게임 내 어뷰징

· 매크로·자동 사냥·봇
· 핵·치트(메모리 변조·게임 클라 패치)
· 버그 악용 (재화 복사·dupe)
· 매칭 어뷰징(de-rank, throwing)
탐지 시그널: 행동 분석, anti-cheat, 인게임 텔레메트리

④ 외부 어뷰징

· RMT(현금 거래)·작업장·골드파밍
· 외부 사이트 거래·계정 거래
· 광고 사기(가짜 리뷰·평점 조작)
· 도용 콘텐츠·해적판 서버
탐지 시그널: 거래 패턴, 외부 모니터링, 신고

계정 탈취 (ATO) · 크리덴셜 스터핑

게임 계정은 가상 재화·결제 수단·소셜 그래프가 모여 있어 ATO 표적 1순위. 다른 사이트에서 유출된 크리덴셜이 게임에 그대로 쓰이는 사례가 가장 흔하다.

탐지 시그널

· 새 IP·새 디바이스·새 국가에서 로그인
· 로그인 직후 인벤토리 이동·아이템 거래·결제
· Velocity (분당 시도 수)
· 비정상 user-agent·known bad IP

방어 1순위 — MFA

2단계 인증이 가장 강력한 단일 통제. 게임 결제·아이템 거래에 step-up MFA 적용(평소엔 묻지 않다가 위험 시점에만). 의무화는 사용자 거부감 — 인센티브로 유도.

크리덴셜 모니터링

외부 유출 DB(HIBP·Recorded Future·자체 dark web 모니터링)와 자사 비밀번호 해시를 비교해 일치 시 강제 재설정. "한 번 유출 = 영원히 차단"이 원칙.

패스키·소셜 SSO

비밀번호 자체를 없애면 ATO도 없다. passkey(WebAuthn)·Apple Sign-in·Google·LINE SSO를 1차 옵션으로. 비밀번호는 2차로 후퇴.

표준 ATO 대응 플레이북

(1) 의심 로그인 차단 + 디바이스 챌린지 (2) 사용자 알림 + "본인이 아니면 비번 변경" CTA (3) 24시간 내 인벤토리 거래·아이템 출고 동결(soft hold) (4) 본인 확인 후 거래 동결 해제 (5) 손실 시 복구 (6) 확정 시 IP·디바이스 ban + 크리덴셜 강제 회전

⚠ PITFALL
ATO 의심 시 곧바로 영구 정지하면 무고한 사용자가 막힌다. "동결(freeze) → 확인 → 해제 또는 정지"의 단계적 대응이 표준. 동결은 짧은 시간(24~72h) 내 풀 수 있어야 한다.

결제 사기 · 환불 어뷰징

결제 사기는 신고가 즉시 매출 손실 + chargeback 수수료 + 스토어 평가 하락으로 이어진다. P3 결제 가이드와 함께 봐야 하지만 어뷰징 관점에서도 별도 다룬다.

도용 카드 (Stolen Card)

훔친 카드로 결제 → 아이템 받고 양도 → 진짜 카드 주인이 chargeback. 도용 카드 BIN·발급국·디바이스 패턴으로 탐지. 신규 계정 + 고액 결제 + 빠른 거래는 강한 시그널.

Friendly Fraud (자기 chargeback)

본인이 결제 후 카드사에 "내가 안 했다"고 환불 받기. 미성년 자녀가 부모 카드로 결제한 케이스도 흔함. 로그·동의·약관 증거가 분쟁 자료.

환불 정책 악용

Steam 2시간/14일 룰을 매번 활용 — 세일·이벤트 콘텐츠를 받고 환불. 반복 환불 사용자에 한도를 두는 것이 Steam의 실제 정책.

국가별 차익 거래

저가 region에서 결제 → 고가 region에서 사용. VPN·결제 카드 발급국 불일치가 시그널. region locking·VPN 차단이 1차 방어.

★ NOTE
결제 어뷰징 탐지는 PG·스토어와 협업 의무가 있다. Apple/Google은 자체 fraud detection을 운영하지만 퍼블리셔 자체 탐지가 없으면 매출의 1~3%가 chargeback으로 사라진다. P3 결제 가이드에 정합성 분석 깊이.

매크로 · 봇 · 자동화 도구

매크로·봇은 게임 도메인 특수 어뷰징. 경제 인플레이션·정상 사용자 경험 저하·서버 부하 모두 영향. 탐지가 가장 어려운 영역 중 하나.

매크로 (외부 자동화)

키보드·마우스 자동화 도구가 게임 외부에서 입력 보냄. 특정 사냥터에서 24시간 반복 → 재화 축적. 입력 패턴(타이밍·궤적·휴식 없음)이 사람과 다름.

봇 (게임 내 자동화)

클라이언트를 직접 조작하거나 패킷을 모방. 매크로보다 정교 — 게임 내 행동(이동·전투·아이템 줍기)이 자동. 의사결정 트리가 단조로워 행동 클러스터링으로 탐지.

탐지 기법

· 입력 통계 분석 (휴식 빈도·반응 시간 분포)
· 행동 시퀀스 ML (Markov chain·LSTM)
· 클라이언트 무결성 (anti-cheat SDK)
· 서버 측 비동기 신호 (CAPTCHA·간단 퍼즐)

제재 단계

1차 경고·짧은 정지 → 2차 장기 정지 + 재화 회수 → 3차 영구 정지. 재화 회수가 매크로 경제의 인센티브를 직접 끊는 가장 효과적인 도구.

⚠ NOTE
한국에서는 매크로·자동프로그램 관련해 형사처벌(게임산업법 §32 등)이 가능한 케이스가 있다. 단순 BAN을 넘어 형사 고소까지 가는 큰 사건도 있어 법무 협업이 필요. 단, 한국에 한정된 접근법이라 글로벌 정책에 그대로 적용 불가.

RMT · 작업장 · 골드파밍

Real-Money Trading (RMT)는 게임 내 재화·아이템·계정을 현금으로 거래. 게임 내 경제를 무너뜨리고 ATO·매크로·결제 사기와 결합되어 있어 어뷰징의 끝판왕.

왜 위험한가

· 인플레이션 — 작업장이 만든 재화가 정상 사용자 경험을 망침
· ATO 수요 창출 — 훔친 계정의 아이템도 거래됨
· 탈세·자금세탁 의심 — 법무 리스크
· 플랫폼 정책 위반 — 스토어 평가 하락 위험

탐지 시그널

· 비정상 거래 패턴 (한 계정에서 다수에게 분산)
· 외부 거래 사이트 모니터링 (자사 게임 매물)
· 채팅·메일 키워드 (외부 사이트 URL·연락처)
· 계정 그래프 분석 (작업장 → 중간 → 사용자)

제재 정책

· 작업장 계정: 영구 정지 + 재화 압수
· 중간 거래자: 정지 + 경고
· 구매자: 1차 경고 + 아이템 회수 → 반복 시 정지
"구매자도 처벌"이 핵심 — 수요를 끊지 않으면 작업장이 계속 생긴다.

대안 — 공식 거래소

일부 게임(EVE Online·디아블로 등)은 공식 거래소를 두어 RMT를 양성화. 단, 한국은 게임 내 재화의 현금 거래가 게임산업법으로 제한적이라 신중한 검토가 필요.

탐지 → 제재 파이프라인

자동 탐지·인간 검토·제재·이의제기를 한 줄기 파이프라인으로 설계해야 한다. EU DSA·AI Act는 사용자 이의제기 권리를 명시 — 인간 검토 없는 자동 정지는 위법 위험.

저위험 자동 / 고위험 사람

매크로 의심 24시간 정지·resource 회수 → 자동. 영구 정지·계정 박탈 → 반드시 사람. 자동/수동 임계는 모델 정밀도와 사용자 영향을 함께 본다.

증거 패키지

제재 결정에는 로그·시그널·정책 인용·결정자가 묶인 패키지가 따라붙는다. 이의제기 시 그대로 재검토 자료로 사용.

이의제기 — 다른 GM 재검토

최초 결정자가 자기 결정을 재검토하면 인용률이 0에 수렴. 반드시 다른 GM이 재검토해야 의미 있음. EU DSA·AI Act 모두 이를 명시.

모델 모니터링

탐지 모델은 자체로 운영 시스템. 거짓 양성률·인용률·드리프트를 매주 본다. 모델이 갑자기 false positive 폭증 → 자동 정지를 일시 중단해야 함.

● Korean Compliance

한국 컴플라이언스 (2026-05 기준)

한국 게임 운영의 컴플라이언스는 게임산업진흥에 관한 법률(게임산업법)과 시행령·고시가 핵심 축. 정책이 자주 바뀌므로 출시·갱신 시점에 항상 공식 자료 재확인이 필수다.

등급분류 — GRAC · GCRB · 자체등급분류

한국에서 게임을 유통하려면 사전 등급분류가 의무. 어디서 받느냐가 게임 유형·플랫폼에 따라 다르다.

기관주관 게임물등급비고
게임물관리위원회 (GRAC)아케이드·모바일·시험용·준사행성, 청소년이용불가 PC·콘솔전체 4단계문체부 산하 기타공공기관

자체등급분류사업자

대표적으로 Apple, Google, Steam(Valve), 닌텐도, 카카오게임즈, 구글플레이 등이 지정되어 있다(시점에 따라 다름). 사업자가 직접 등급분류 → GRAC에 결과 통보.

실무 시나리오

· 모바일 게임 → Apple/Google이 자체등급분류 + GRAC 통보
· Steam PC 게임 → Valve가 자체등급분류 (청소년이용불가는 GRAC 직접)
· 자체 PC 런처 → GCRB 또는 GRAC 직접

등급분류 신청 자료

게임 콘텐츠 영상·기능 명세·확률형 아이템 정보·결제 시스템·청소년 보호 시스템 등. "어떤 콘텐츠가 어떻게 노출되는지"를 객관적으로 입증해야 함.

위반 시 영향

등급분류 받지 않은 게임은 유통 금지. GRAC 직권 등급조정 가능. 광고에서도 등급 표시 의무. 위반 시 형사처벌까지.

게임시간 선택제

게임산업법 제12조의3. 만 18세 미만 청소년 본인 또는 법정대리인의 요청 시 게임 이용 시간을 설정할 수 있는 제도. 강제적 셧다운제는 2022-01-01에 폐지되어 선택제로 일원화.

현재 운영 (2026-05)

· 청소년 본인 또는 법정대리인이 요청 시 시간대 설정
· 게임사는 요청에 응해야 함 (의무)
· 강제적 셧다운제(밤 12시~새벽 6시 차단)는 2022-01-01 폐지되어 선택제로 일원화

구현 요건

· 본인 또는 법정대리인 인증 절차
· 요청한 시간대에 자동 차단
· 변경·해제도 본인/법정대리인 인증 필요
· 운영 기록 보존

2024-2028 정책 동향

「게임산업 진흥 종합계획(2024~2028)」에 게임시간 선택제 의무 조항을 폐지하고 게임사가 자체적으로 운영하도록 개정 계획. 시점은 미정 — 출시 시점에 공식 공지 재확인 필요.

실무 팁

법령 변경 가능성이 있어도 현재 의무 조항은 유효. 시스템에서 빼면 안 된다. 정부 종합계획대로 폐지되면 그때 자율 운영으로 전환.

확률형 아이템 표시 의무

2024-03-22부터 시행. 게임산업법 제33조 + 시행령 별표3의2(2024-01-09 신설)가 표시 내용·방법을 정의. 22대 국회에서 위반 시 3배 배상·입증책임 전환을 추가 강화하는 개정안 발의 진행 중.

표시 대상

· 일반 확률형 아이템 (단순 가챠)
· 제비뽑기형 (10연차 등 묶음)
· 강화형 (아이템 강화 성공률)
· 천장(pity) 시스템이 있는 경우 천장 횟수·확률
· 시행령 별표3의2가 유형별 표시 사항을 명시

표시 위치 의무

· 게임 클라이언트(가챠 화면 또는 인접)
· 게임 공식 웹사이트
· 광고·홍보물 (광고에는 "확률형 아이템 포함" 등 표시)
한 곳만 누락해도 위반

변경 의무

확률을 바꾸면 변경 즉시 표시 갱신 + 변경 사실 공지. 비밀리에 너프하는 행위는 강한 위반.

위반 시

현행: 게임위 시정명령 + 영업정지 가능 + 손해배상 청구 가능. 22대 국회 발의안에 따라 향후 3배 배상·입증책임 전환이 추가될 수 있음.

⚠ PITFALL
"표시는 했는데 실제 확률과 다르다" 케이스가 가장 큰 리스크. RNG 코드와 표시 데이터의 single source of truth를 두고 자동 검증해야 한다. 확률 변경은 코드 변경 + 표시 갱신이 atomic이어야 함.

청소년 결제 · 한도

게임 결제 한도와 청소년 결제 동의는 시기·정책별로 다르다. 2026-05 기준 운영을 정리.

성인 결제 한도

2019-07 게임산업법 시행령 개정으로 PC 온라인 게임 월 50만원 한도 폐지. 모바일은 별도 한도가 명시적으로 없음. 단, 자율규제로 일부 게임이 자체 한도 운영.

청소년 결제

· 청소년이용가 게임도 법정대리인 동의가 필요
· 청소년이용불가 게임은 청소년이 결제 불가
· 모바일은 스토어가 자체 청소년 보호 (Family Sharing 등)

본인인증

한국 게임은 사실상 본인인증(휴대폰·아이핀·금융인증)이 보편. 청소년이용불가는 본인 확인 의무. 청소년이용가도 결제 시점에 본인 확인이 일반적.

실무 팁

스토어 결제(Apple/Google/Steam)는 스토어가 청소년 보호를 1차 처리. 퍼블리셔는 그 위에 추가 게이팅을 둘 것인지 자율 결정. 청소년이용가에서도 자체 결제 한도를 두는 게 일반적.

게임 광고 규제

게임 광고는 표시광고법·청소년보호법·게임산업법의 교차점. 위반 시 게임위·공정위·방심위 등 다수 기관이 동시 관할.

등급 표시 의무

광고에 게임 등급 표시 (전체이용가·12세·15세·청소년이용불가). 영상 광고 끝에 등급 마크 + 확률형 아이템 포함 시 표시.

허위·과장 광고 금지

표시광고법. 실제 게임에 없는 콘텐츠를 광고에 보여주는 것은 위반. 사전 녹화 영상이 실제 게임플레이와 다르면 분쟁 소지.

가챠 표시

광고에 확률형 아이템이 포함되어 있다는 사실 + 확률 정보(또는 게임 내 확률 안내 페이지 안내) 표시 의무.

청소년 유해성

청소년이용불가 게임을 청소년이 보는 매체에 광고하면 위반. 시간대·매체별 게이팅이 필요.

★ NOTE
한국 게임 광고는 한국게임산업협회 자율심의기구를 통한 자율심의가 일반적. 자율심의 통과한 광고는 분쟁 시 1차 방어선이 됨. 사전 심의 받지 않은 광고는 사후 문제 발생 시 즉시 중단·정정 의무.
● Global Compliance

글로벌 컴플라이언스

글로벌 게임 운영은 EU·미국·일본·중국·동남아·중동의 규제를 동시에 만족해야 한다. 2026년 현재 EU의 DSA·AI Act, 일본의 APPI 개정, 중국의 판호 재개 등이 가장 큰 변동 요인.

EU GDPR — 데이터 보호의 기본축

2018년 시행된 EU 일반개인정보보호규정. 글로벌 운영의 사실상 표준. 한국 PIPA·일본 APPI도 GDPR을 의식한 개정이 진행 중.

핵심 원칙

· 처리 근거 명확화 (동의·계약·법적 의무·정당한 이익)
· 목적 제한·최소 수집
· 사용자 권리 (열람·정정·삭제·이동·이의제기)
· 데이터 처리자(Processor) 책임 분리
· DPO(Data Protection Officer) 지정 (대규모)

국제 이전

EU 외 이전은 SCC(Standard Contractual Clauses) 또는 적정성 결정·BCR. 한국은 2021년 일부 적정성 결정. 미국은 EU-US Data Privacy Framework. 중국·러시아는 별도 계약·법적 검토 필수.

아동 동의

16세 미만은 부모 동의 (회원국별로 13~16세). 게임은 회원가입·인앱 결제·광고 추적 모두 영향. 나이 인증 메커니즘이 핵심.

위반 과징금

최대 글로벌 매출 4% 또는 €20M 중 큰 금액. 실제 부과 사례 多(Meta·Amazon·구글). "잊혀질 권리" 미응답이 가장 흔한 위반.

미국 — COPPA · CCPA · CPRA

미국은 연방 단일 법이 없고 주별로 다르다. 캘리포니아 CCPA/CPRA가 사실상 표준이고 그 위에 COPPA(아동)·VPPA(영상)·HIPAA(건강) 등 세부 법이 겹친다.

COPPA (Children's Online Privacy Protection Act)

13세 미만 아동 대상 서비스에 부모 동의·정보 최소화·삭제권. "13세 미만"임을 알면서도 데이터를 수집하면 큰 위반. FTC가 강력히 집행.

CCPA / CPRA

캘리포니아 거주자 대상. 사용자에게 데이터 조회·삭제·판매 거부권 부여. "Do Not Sell or Share My Personal Information" 링크 의무.

주별 신규 법

버지니아(VCDPA), 콜로라도(CPA), 코네티컷(CTDPA), 텍사스(TDPSA), 유타(UCPA), 오리건, 델라웨어 등이 자체 법 시행. 2026년 현재 20+개 주가 자체 법 보유.

실무 — 단일화 전략

주마다 별도 구현은 비현실적. 가장 강한 주(CA) 기준으로 단일화 + 주별 미세 조정이 일반적. EU GDPR과 합쳐서 "통합 동의·삭제 시스템" 운영.

일본 APPI — 2026 개정 동향

개인정보 보호법(APPI). 2003년 제정 후 정기 개정. 2026-04-07 일본 내각이 새 개정안을 승인하여 국회 제출, 공포 후 2년 이내 시행 예정.

2026 개정 핵심 — 완화

AI 학습·통계 처리에 대한 동의 면제 신설. 공개된 민감 정보 수집·제3자 제공도 투명성·계약 안전장치 조건 하에 허용. 일본의 데이터 활용 활성화 정책.

2026 개정 핵심 — 강화

· 16세 미만 아동 데이터 처리에 부모 동의 명시
· 행정 과징금 제도 도입
· 적격 소비자 단체에 의한 금지 청구·손해배상 청구 제도

현행 의무 (2026-05)

· 동의·이용 목적 통지
· 안전관리조치 (보안)
· 개인정보 취급사업자 의무
· EU와 상호 적정성 인증 (2019)

실무 영향

한국 게임의 일본 운영은 GDPR 수준 대비 약간 가벼웠던 게 사실. 2026 개정 후에는 GDPR과 거의 동일한 수준의 시스템 정비가 필요해질 가능성.

중국 판호 (版号) — 2026 발급 동향

중국에서 게임을 상용 출시하려면 국가신문출판서(NPPA)의 판호가 필수. 2021년 압박 → 2024-2025 재개 → 2026 본격 회복 단계.

판호의 종류

· 내자판호 — 중국 자국 게임용
· 외자판호 — 해외 게임 수입용 (한국 게임은 여기 해당)
발급권자: 국가신문출판서(NPPA, 国家新闻出版署)

2025-2026 추이

2025년 누적 판호 1,771개 (전년 대비 20% 증가, 2019년 이후 최고치). 2025-12 발표 147종 중 외자 3종 포함. 한국 게임의 외자판호 획득이 점차 증가 추세.

운영 요건

· 중국 현지 퍼블리셔 파트너 필수 (NPPA 신청 자격은 중국 법인)
· 콘텐츠 검열 (정치·역사·종교·동성애·과도한 폭력 등 제한)
· 데이터는 중국 내 보관 (Cybersecurity Law)
· 미성년 게임 시간 제한 (주말 1시간 등)

2026 정책 기조

2021년 "정신적 아편" 압박에서 "디지털 경제 핵심 성장 동력"으로 재평가. 단, 정치 변동에 민감 — 출시 시점에 NPPA 공식 공지 재확인 필수.

EU DSA · DMA — 게임 플랫폼 영향

Digital Services Act(2024 발효, 2026-02 시행 2주년). 게임 플랫폼·앱스토어·게임 마켓플레이스에 모두 적용. "제재 시 사용자 권리 보장"이 핵심.

적용 범위

· 모든 온라인 플랫폼·앱스토어
· EU 사용자 4500만+ → VLOP(Very Large Online Platform) 지정 (Apple App Store·Google Play·Steam 등)
· VLOP은 가장 강한 의무

핵심 의무

· 불법 콘텐츠 신속 제거
· 제재 시 사용자에게 사유·이의제기 절차 안내 (Statement of Reasons)
· 광고 투명성 (누가·왜 본인에게 광고하는지)
· 알고리즘 투명성 (추천·우선순위 작동 방식)

VLOP 추가 의무

· 시스템 위험 평가·완화
· 외부 감사 (independent audit)
· 데이터 접근 제공 (연구자·당국)
· 위기 대응 메커니즘 (재해·선거)

위반 과징금

글로벌 매출 6% (GDPR보다 강함). 실제 부과 시작됨. "숨은 알고리즘"이 가장 큰 리스크 — 추천·랭킹·매칭 메커니즘 문서화 필수.

⚠ NOTE
퍼블리셔 자체가 VLOP에 해당하기 어렵지만, VLOP에 게임을 올리는 입장에서는 스토어의 DSA 의무에 협조해야 한다. 예: 스토어가 콘텐츠 신고를 받으면 게임사에 자료 요청 — 응답 의무가 계약상 강화되는 추세.

EU AI Act — 게임 AI · Anti-cheat 영향

2024년 발효 후 단계적 시행. 2025-02-02 prohibited 시행, 2026-08-02 high-risk 본격 시행. 위반 시 €35M 또는 글로벌 매출 7%.

리스크 분류

· Prohibited: 조작·사회 점수·집단 생체 식별·민감 영역 감정 인식
· High-risk: 채용·교육·신용·법 집행 등
· Limited risk: 챗봇·딥페이크 (투명성 의무)
· Minimal risk: 게임 AI 대부분

게임은 일반적으로 minimal-risk

일반 게임 AI(NPC·매칭·anti-cheat 모델)는 minimal-risk로 분류된다는 게 EU 공식 입장. 특별한 의무 없음. 단 prohibited 기법 사용 시 즉시 위반.

주의해야 할 prohibited

· 정서·감정을 조작해 결제 유도 (다크 패턴)
· 동의 없는 생체 인식·감정 인식
· 사회 점수식 사용자 평가
이런 기법을 사용하면 게임 AI도 prohibited 영역으로.

Limited risk — 챗봇·생성형

게임 내 LLM·생성형 AI는 "AI와 대화 중"임을 사용자에게 알릴 의무. 생성된 콘텐츠는 AI 생성물임을 식별 가능하게 표시.

★ NOTE
game AI는 거의 minimal-risk이지만, 프로파일링 기반 결제 유도·LLM 챗봇·생성형 NPC는 limited-risk 의무가 적용된다. AI Act 시행 직전(2026-08)에 모든 게임의 AI 사용 매트릭스를 점검하는 게 표준 작업.
